# 投递流程

本文说明 `guoqi-job-hunter` 中哪些步骤可以自动化，哪些步骤必须由用户手动确认。

## 可以自动化的部分

* 监控公开招聘网站、招聘公告页和校招入口。
* 提取公告标题、单位、岗位、地点、报名时间、专业要求、学历要求和报名链接。
* 根据通信工程专业、CET4、计算机二级、供电所实习经历进行岗位匹配评分。
* 对国家电网、南方电网、三大运营商、央国企技术岗及其子公司进行投递优先级排序。
* 导出岗位清单 Excel 或 CSV。
* 生成投递提醒、简历优化建议和面试准备建议。

## 必须人工确认的部分

* 登录招聘系统账号。
* 处理验证码、短信验证、人脸识别或其他身份校验。
* 选择最终投递岗位、志愿顺序和工作地点。
* 上传、替换或确认最终简历版本。
* 检查姓名、电话、邮箱、学校、专业、证书、实习经历等个人信息。
* 点击“最终提交”“确认投递”“报名确认”等不可撤销按钮。

## 禁止自动化的部分

* 不要绕过验证码。
* 不要破解、模拟或规避招聘系统安全机制。
* 不要自动提交最终申请。
* 不要伪造证书、成绩、实习经历或个人身份信息。
* 不要在用户未确认的情况下修改志愿顺序或岗位选择。

## 推荐流程

1. 每天或每周运行监控脚本，生成候选公告。
2. 使用匹配模块筛选通信工程、网络运维、信息化、电力通信、云网、技术支持相关岗位。
3. 导出 Excel，按匹配评分和投递优先级排序。
4. 用户人工打开报名链接，核对公告原文和岗位要求。
5. 根据简历优化建议调整简历，但不夸大或伪造经历。
6. 用户人工登录招聘系统并处理验证码。
7. 用户人工上传简历、选择岗位、确认个人信息。
8. 最终提交前再次检查岗位、地点、联系方式、附件和报名截止时间。
9. 用户确认无误后，手动点击最终提交。

## 当前版本边界

当前版本优先实现：

* 监控
* 筛选
* 匹配评分
* Excel/CSV 导出
* 提醒信息生成

后续版本再扩展半自动投递辅助，例如：自动打开报名页面、自动填充草稿信息、生成待确认投递清单。但任何最终提交动作仍必须由用户人工确认。
